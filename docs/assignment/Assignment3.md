# CS336 Spring 2024 Assignment 3: Scaling Laws - 详细总结

## 1. 作业概览 (Assignment Overview)

### 1.1 核心目标
本作业旨在让学生**亲手实践并构建大语言模型（LLM）的缩放定律（Scaling Laws）**。学生将扮演 ClosedAI 首席科学家的角色，在有限的计算预算（FLOPs）下，通过权衡**模型大小（Model Size, $N$）**与**训练数据量（Training Tokens, $D$）**，寻找能产生最低训练损失的“计算最优”（Compute-Optimal）模型配置。

### 1.2 任务背景
*   **场景**: 训练下一代 LLM，计算成本极高（相当于一个小国的 GDP），因此必须一次性做出最优决策。
*   **约束**:
    *   **目标预算**: $1 \times 10^{19}$ FLOPs（用于最终预测）。
    *   **实验预算**: $2 \times 10^{18}$ FLOPs（仅占目标预算的 20%，用于拟合缩放定律）。一旦超出此限制，API 将拒绝请求。
*   **工具**: 使用提供的**训练 API**查询不同超参数配置下的最终训练损失，而非亲自训练模型。
*   **核心挑战**: 如何在小规模实验数据的基础上，准确外推（Extrapolate）到大规模预算下的最优配置。

### 1.3 提交要求
1.  **报告 (`writeup.pdf`)**: 详细描述拟合缩放定律的方法论、实验设计思路、数据拟合过程及最终预测结果。需具备可复现性。
2.  **代码 (`code.zip`)**: 包含所有用于查询 API、处理数据、拟合曲线及预测的代码。
3.  **在线表单提交**: 提交预测的**最优模型参数量**、**训练超参数**（层数、维度等）及**预测损失值**。这部分表现将直接计入成绩。

## 2. 理论基础：缩放定律回顾 (Scaling Laws Review)

### 2.1 核心问题
给定计算预算 $C$，如何选择超参数（模型大小 $N$、训练 token 数 $D$ 等）以最小化训练损失 $L$？
*   **计算公式**: 训练 Transformer 的近似计算量为 $C \approx 6ND$。
*   **权衡**: 在固定 $C$ 下，增大 $N$ 必然导致 $D$ 减小，反之亦然。存在一个最优平衡点。

### 2.2 IsoFLOPs 方法 (Hoffmann et al., 2022 / Chinchilla)
这是作业推荐的核心方法论：
1.  **IsoFLOPs 剖面 (Profiles)**: 对于一系列不同的计算预算 $C_i$，分别训练多个不同大小 $N$ 的模型（此时 $D = C_i / 6N$）。
2.  **寻找最优值**: 对于每个固定的 $C_i$，观察损失 $L$ 随 $N$ 的变化。
    *   $N$ 过小：模型容量不足，损失高（欠拟合）。
    *   $N$ 过大：训练步数不足，损失高（未收敛）。
    *   **最优解 ($N_{opt}$)**: 在该预算下损失最低的模型大小。经验上，$L$ 与 $N$ 在固定 $C$ 下呈二次关系。
3.  **拟合幂律 (Power Laws)**: 收集所有 $\langle C_i, N_{opt}(C_i) \rangle$ 和 $\langle C_i, D_{opt}(C_i) \rangle$ 数据点，拟合幂律关系：
    *   $N_{opt} \propto C^a$
    *   $D_{opt} \propto C^b$
4.  **外推**: 利用拟合出的幂律，预测目标大预算（如 $10^{19}$ FLOPs）下的最优 $N$ 和 $D$。

### 2.3 其他参考
*   **Kaplan et al. (2020)**: 早期缩放定律，主要关注 $N$ 和 $D$ 对损失的独立影响。
*   **Yang et al. (2022)**: 关于超参数零样本迁移的研究，可辅助确定学习率等超参数。

## 3. 核心任务：构建缩放定律 (Constructing Scaling Laws)

### 3.1 训练 API 使用指南
学生需通过 HTTP GET 请求查询 API，获取模拟训练的最终损失。
*   **接入方式**: 需连接斯坦福网络（或使用 VPN），使用个人 SSH Public Key 作为 `api_key`。
*   **关键端点**:
    *   `GET /loss`: 输入实验配置，返回 `loss` 和当前累计使用的 `total_flops_used`。
    *   `GET /total_flops_used`: 查询当前已消耗的 FLOPs 预算。
*   **可调节超参数范围**:
    *   `d_model`: [64, 1024]
    *   `num_layers`: [2, 24]
    *   `num_heads`: [2, 16]
    *   `batch_size`: {128, 256}
    *   `learning_rate`: [1e-3, 1e-4]
    *   `train_flops`: 离散集合 {1e13, ..., 1e18}
*   **注意**: 重复查询相同配置不消耗额外预算。

### 3.2 实验设计与策略 (关键得分点)
由于预算极其有限（$2 \times 10^{18}$），必须精心规划实验点：
1.  **预算分配**:
    *   如何分布不同的 $C_i$ 值？（建议覆盖多个数量级，如 $10^{14}$ 到 $10^{17}$）。
    *   在每个 $C_i$ 下，采样多少个不同的 $N$？（需足够多以捕捉二次曲线的最低点）。
2.  **超参数控制**:
    *   除 $N$ 和 $D$ 外，其他超参数（如学习率、Batch Size）如何设定？
    *   建议先在小规模下探索学习率对损失的影响，或在不同 $C_i$ 下动态调整学习率（参考 Chinchilla 或 Kaplan 的建议）。
3.  **异常处理**:
    *   识别并剔除因超参数设置不当（如学习率过大导致发散）而产生的无效数据点。

### 3.3 数据拟合与外推
1.  **确定 $N_{opt}(C_i)$**: 对于每个预算 $C_i$，从实验数据中找出损失最小的运行点，记录其 $N$ 和 $D$。
    *   *提示*: 作业建议直接取最小损失点，而非拟合二次曲线求极值，以简化流程。
2.  **拟合幂律**:
    *   使用 `scipy.optimize.curve_fit` 或其他回归方法，在双对数坐标（Log-Log Plot）下拟合直线。
    *   模型形式：$\log(N_{opt}) = a \cdot \log(C) + k_1$，$\log(D_{opt}) = b \cdot \log(C) + k_2$。
3.  **评估拟合质量**:
    *   绘制散点图与拟合曲线，检查残差。
    *   讨论拟合的置信度及潜在偏差。

### 3.4 最终预测 ($10^{19}$ FLOPs)
1.  **计算最优配置**: 将 $C = 10^{19}$ 代入拟合好的幂律公式，计算预测的 $N_{pred}$ 和 $D_{pred}$。
2.  **映射回具体超参数**:
    *   根据公式 $N \approx 12 \cdot n_{layer} \cdot d_{model}^2$（忽略 Embedding 参数），反推具体的 `num_layers` 和 `d_model`。
    *   确保推导出的参数在 API 允许的范围内（若超出，需讨论截断或外推风险）。
    *   确定配套的 `num_heads`, `batch_size`, `learning_rate`。
3.  **预测损失**: 基于缩放定律公式 $L(N, D) = E + \frac{A}{N^\alpha} + \frac{B}{D^\beta}$ 或直接利用趋势外推预测最终 Loss。

## 4. 训练运行细节 (Training Run Details)

了解 API 背后的模拟环境对于正确解释数据至关重要：
*   **模型架构**: 标准 Decoder-only Transformer（类似 Assignment 1），但使用 **LayerNorm** (非 RMSNorm) 且 **Embedding 未绑定** (Untied)。
*   **参数量估算**: $N \approx 12 \cdot n_{layer} \cdot d_{model}^2$。
*   **数据集**: SlimPajama，使用 32K vocab 的 Byte-level BPE Tokenizer。
*   **上下文长度**: 512。
*   **FFN 维度**: $4 \times d_{model}$。
*   **正则化**: Attention 和 Residual Dropout 概率均为 0.1。
*   **优化器**: AdamW (Weight Decay=0.01, Grad Clip=1.0)。
*   **学习率调度**: Cosine Annealing (无 Warm-up)，从初始 LR 衰减 10 倍，周期等于总训练步数。

## 6. 注意事项与技巧 (Tips & Best Practices)

### 6.1 预算管理与调试
*   **本地模拟**: 在调用真实 API 前，先用少量数据或硬编码测试代码逻辑，避免浪费宝贵的 FLOPs 预算在调试上。
*   **监控用量**: 每次查询后务必检查 `total_flops_used`，防止意外超支。
*   **缓存结果**: 将 API 返回结果保存到本地 JSON，避免重复查询（虽然不扣费，但节省时间）并方便断点续传。

### 6.2 实验设计策略
*   **对数均匀采样**: 在 $C$ 和 $N$ 的空间上，建议采用对数均匀分布采样，以覆盖更广泛的尺度。
*   **密集采样关键点**: 在预计的最优点附近（通常 $N$ 与 $C$ 的某种比例关系处）适当增加采样密度，以提高拟合精度。
*   **学习率敏感性**: 注意 Chinchilla 指出最优学习率随 $D$ 变化。在不同 $C_i$ 下，可能需要根据 $D$ 调整 LR，否则得到的 $N_{opt}$ 可能会有偏差。

### 6.3 常见陷阱
*   **参数范围限制**: API 对 `d_model` (max 1024) 和 `num_layers` (max 24) 有限制。如果外推结果超出此范围，需讨论如何处理（例如：是否意味着最优模型超出了 API 的模拟能力？或者需要调整架构比例？）。
*   **过拟合噪声**: 模拟数据可能包含随机噪声。不要过度拟合单个异常点，应关注整体趋势。
*   **单位换算**: 注意参数量 $N$ 的计算公式，确保与 API 配置的 `d_model` 和 `num_layers` 对应准确。

### 6.4 资源与工具
*   **Python 库**: `requests` (API 调用), `numpy`, `matplotlib` (绘图), `scipy` (曲线拟合), `json` (数据处理)。
*   **参考文献**: 深入阅读 Kaplan et al. (2020) 和 Hoffmann et al. (2022) 原文，理解其实验设置和拟合公式的具体形式。
